## Why this document exists

The Harmony Framework and Constella were discovered to be structurally isomorphic
*mid-build* — the body-resonance architecture and the civic architecture kept producing
the same shapes ("I didn't realize that I was somewhat building it like that", Dec 10).
This bridge is what makes `harmony/` a part of Constella rather than a lodger in its
repo: one architecture, two substrates. Harmony models how a body processes
gravitational load without rigid structure; Constella models how a polity processes
accountability load without rigid hierarchy.

## The mapping

| Harmony (body) | Constella (civic) | Shared principle |
|---|---|---|
| Gravitational load | Accountability load (obligations, disputes, civic work) | Systems are defined by what they must continuously bear, not what they own |
| Head — first-stage gain, orientation | Civic perception — how citizens receive and orient to collective signals | Input is *interpreted*, never raw |
| **Phase flip** (tension → stabilization) | **Penumbra Accord** — dissent transformed into constructive accountability | Opposition is load-bearing, not failure; the flip point is where energy changes role |
| Yang intake (outward, receiving) | **Astris** — participation, outward contribution | Intake must be structurally rewarded or the system starves |
| Yin return (grounding, consolidating) | **Auctor** — accountability, grounding responsibility | Every intake needs a return path; feedforward-only systems drift (see Harmony-AI Bridge: the missing return loop in current AI) |
| Yin return as **rest / restoration** (recovery phase, not only accountability) | **Temporal Floor** — guaranteed fallow time (added 2026-07-10) | The return phase must be *protected*, or continuous intake consumes the system — a feedforward-only polity (all work, all attention-capture) drifts and collapses |
| Phase-flip **rhythm** — the recurrence of intake↔return over time | **Governance Cadence** — the published, loose, multi-zone clock (added 2026-07-10) | A coherent *shared* rhythm lets the polity phase-flip cheaply and anticipate (cf. ALife `entrainment`); no rhythm → thrash, one rigid clock → homogenize (cf. `niche-maintains-diversity`) |
| Feet — second-stage gain, ground contact | Local governance / community implementation | Abstract load must eventually meet concrete ground; that contact point needs its own gain stage |
| Hands — micro-stabilizers | Civic gestures — voting, testimony, mutual aid, deliberation | Fine-tuning happens through many small, reversible acts, not rare large ones |
| Micro-buoyancy (dynamic equilibrium via continuous micro-adjustment) | Universal Civic Floor — continuous small supports instead of episodic rescue | Stability is maintained, never achieved |
| Fascial continuity (load transfer through connected tissue) | Federated structures / coalition fabric | Load distributes only through *connected* networks; isolation concentrates stress |
| Cross-system prediction (jaw change → measurable subtalar change) | Policy tremor — a governance change at one node produces measurable effects at distant nodes | The isomorphism's empirical edge: both claim distant coupling is real and testable |

## What the bridge is for (practical uses)

1. **Design checklist**: when adding a Constella mechanism, ask which Harmony element
   it corresponds to. No correspondence → suspect the mechanism is rigid-structure
   thinking (hierarchy) smuggled into a resonance architecture.
2. **FAITHH reasoning layer**: the AI logic stack inherits the same shape — intake
   (perception), phase flip (contradiction handling — cf. confirmability precedence
   rule), return loop (output-coherence sensing), micro-adjustment (continuous small
   corrections over rare large retrainings). See harmony_ai_bridge_v1.0.0.md.
3. **FGS coalition pilot**: if Floating Garden Soundworks becomes a Constella pilot,
   this table is the rubric — a small coalition needs the phase-flip (conflict →
   accountability) and micro-stabilizer (many small acts) mechanisms first, not tokens.

## Boundaries (what this mapping does NOT claim)

- No anatomical claim supports a governance claim or vice versa — the isomorphism is
  structural, and each side stands on its own evidence (Harmony Layer 1 on anatomy;
  Constella on pilots and the founding hypothesis, which remains an honest assumption).
- The mapping is falsifiable as *design guidance*: if a Constella mechanism with no
  Harmony correspondent consistently outperforms mapped ones in pilots, the bridge
  weakens. Track this in pilot retrospectives.
