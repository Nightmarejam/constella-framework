## Math & Mechanics


### 1) Astris (A) ‚Äî Accrual & Decay
- **Accrual per task:** `A_gain = B * C_q * V_p * R_s`
  - `B` = base bounty for task
  - `C_q` = contribution quality (0.6‚Äì1.4 via rubric)
  - `V_p` = verified proof factor (0.8‚Äì1.2; repo links, QR/NFC logs)
  - `R_s` = redundancy score (1.0 if unique; 1.1 if verified backup role)

- **Activity decay (weekly, applied on Sundays 00:00 UTC):**
  - If active (‚â•1 verified action/week): **no decay**.
  - If inactive `w` weeks: `A_new = A_prev * (1 - d)^w`, with `d = 0.02` (2%/week).
  - **Plateau protection:** contributions flagged ‚Äúfoundational‚Äù (>95th percentile impact) are capped at **max 10% total decay** lifetime.

**Example:** A=1000, 3 inactive weeks ‚Üí `1000*(0.98^3)=941.192`.

### 2) Auctor (Au) ‚Äî Equality Vote with Light Decay
- **Base grant:** 1 Au per verified person (proof-of-personhood).
- **Inactivity decay:** if no governance participation for `q` quarters:
  `Au_new = Au_prev * (1 - 0.05)^q` (5%/quarter), floor = 0.5.
- **Reactivation:** any two actions in a quarter (vote, review, meeting) restore to pre‚Äëdecay level up to 1.0.

### 3) Cross‚ÄëVault Rebalancing
- **Signal:** vault coverage ratio `CR = Reserves / (3*AvgMonthlyOutflow)`
- **Action:** if `CR < 0.8` for 2 consecutive weeks, auto‚Äëproposal to move up to **min(5%, gap)** from vaults with `CR > 1.2` (subject to Auctor vote + Astris weighted advisory).
- **Guardrail:** single move cannot drop any donor vault below `CR=1.0`.

### 4) Civic Floor (UCF) Allocation
- **Sources:** (a) Vault surpluses ‚â• CR1.5, (b) OPP (Open Palm) pledges, (c) Genesis interest, (d) partner grants.
- **Monthly allocation:** `UCF = min( BasePop*H , 0.25*(Surplus_total) )`
  - `H` = per‚Äëcapita floor (pilot default: **$200** equiv/mo in goods/services)
- **Priority:** food, shelter, utilities, transit, basic care; **in‚Äëkind first**, cash last.

## Verification, Privacy & Anti‚ÄëGaming


- **Local-first ledger:** QR/NFC app stores signed events on device; syncs hashed proofs to network.
- **Zero-Knowledge Badge (ZKB):** venue verifies ‚ÄúMember-in-good-standing‚Äù without seeing PII; rotating keys every 30 days.
- **Anomaly detection:** heuristics on bursty contributions, device reuse, geo‚Äëimpossible overlaps; human steward review escalations.
- **Dual attestation:** risky tasks (PSL2‚Äì3) require **two independent verifications** (device + peer; or device + photo+EXIF redacted).
